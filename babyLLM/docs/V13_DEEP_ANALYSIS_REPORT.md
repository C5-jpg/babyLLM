## 执行摘要

V13 是 ChineseBabyLM 项目截止当前的 SOTA 架构，在 1 亿中文字符数据集上从头预训练，历经 3 阶段训练管线（CLM → MNTP → Polish），最终选出 Stage 2 EMA 模型作为提交版本。

### 核心结论

| # | 结论 | 数据支撑 |
|---|------|---------|
| 1 | **PPL 改善已趋饱和** — V12→V13 仅改善 0.26 PPL 点（38.94→38.68），边际收益急剧递减 | tokens/param 比仅 1.1×，远低于理想 20-200× |
| 2 | **EMA 是最有效的正则化工具** — Stage 1 EMA 带来 8.4% PPL 改善，Stage 3 仅 0.8%，收益递减明显 | eval_stage1.json vs eval_stage1_ema.json |
| 3 | **零样本语法能力薄弱** — ZhoBLiMP 平均 63.47%，3 个维度低于随机基线（anaphor 35%, passive 37%, npi_licensing 46.67%） | 官方评测管线输出 |
| 4 | **微调表现分化** — AFQMC 69.0%（接近基线 70.2%），TNEWS 仅 53.9% | finetune 评测结果 |
| 5 | **Stage 2 EMA 优于 Stage 3** — 尽管 Stage 3 训练了更多步骤，但 Stage 2 EMA 的 PPL (38.68) < Stage 3 EMA (40.08) | eval_stage2_ema.json, eval_stage3_ema.json |

### 1.2 三阶段训练管线设计

```
┌─────────────────────────────────────────────────────────────────────┐
│  Stage 1: CLM Pretraining (SGDR + Focal Loss + EMA)                │
│  ├─ 目标: 标准因果语言模型预训练                                      │
│  ├─ 8 epochs, lr=6e-4, focal_gamma=1.5                             │
│  ├─ SGDR 调度器 (T_mult=2), attention_dropout=0.1                  │
│  └─ 输出: best_model + best_model_ema                              │
│      ↓                                                              │
│  Stage 2: MNTP + Dynamic CLM (Cosine + Focal Loss + EMA)           │
│  ├─ 目标: Masked Next-Token Prediction 混合训练                      │
│  ├─ 10 epochs, lr=5e-4, focal_gamma=1.0                            │
│  ├─ Dynamic CLM ratio: 25%→12.5%→6.25%                            │
│  ├─ Mask ratio: 25%→10% (线性退火)                                  │
│  └─ 输出: best_model + best_model_ema                              │
│      ↓                                                              │
│  Stage 3: Polish (DropBlock + Stochastic Depth)                     │
│  ├─ 目标: 低学习率精细调整 + 结构化正则化                              │
│  ├─ 5 epochs, lr=2e-5, 无 Focal Loss                               │
│  ├─ DropBlock=0.1 (block_size=3), StochDepth=0.05                  │
│  └─ 输出: best_model + best_model_ema                              │
└─────────────────────────────────────────────────────────────────────┘

最终选择: Stage 2 EMA 模型 (PPL 38.68 < Stage 3 EMA 40.08)
```

### 2.2 学习率调度策略分析

#### 2.2.1 各阶段 LR 策略对照

| 阶段 | 调度器 | 初始 LR | 最终 LR | Warmup | 设计意图 |
|------|--------|---------|---------|--------|---------|
| Stage 1 | SGDR (T_mult=2) | 6e-4 | 3.0e-4* | 5% | 周期性重启探索损失面 |
| Stage 2 | Cosine | 5e-4 | 4.8e-5 | — | 标准退火收敛 |
| Stage 3 | Cosine | 2e-5 | 1.1e-5 | — | 极低 LR 精细调整 |

*Stage 1 最终 LR 3.0e-4 是 SGDR 周期内的值（处于重启后的上升段）

#### 2.2.2 SGDR 调度器有效性分析

Stage 1 使用 SGDR (Stochastic Gradient Descent with Warm Restarts)，配置为 T_0=auto（1 epoch = 2804 步），T_mult=2。这意味着：
- 第 1 个余弦周期: step 0→2804
- 第 2 个余弦周期: step 2804→8412 (2804+5608)
- 第 3 个余弦周期: step 8412→19628 (8412+11216)

从训练曲线看，Epoch 5 的 Train Loss 反弹（3.89→4.05）恰好对应 SGDR 的 LR 重启点，说明 **SGDR 成功触发了损失面的重新探索**，最终在 Epoch 7 找到了更好的局部最优。

#### 2.2.3 学习率递减策略评估

V13 的 LR 策略为 6e-4 → 5e-4 → 2e-5，呈现 **阶梯式递减**：

```
LR (log scale):

Stage 1: 6e-4 ██████████████████████████████████████ (SGDR 周期波动)
Stage 2: 5e-4 ████████████████████████████████████ (Cosine 平滑衰减)
Stage 3: 2e-5 ████ (极低 LR)

衰减比: S1→S2 = 0.83×, S2→S3 = 0.04× (S2→S3 的跳变过大)
```

**诊断**: Stage 2→Stage 3 的 LR 跳变（5e-4→2e-5，25倍衰减）可能过于激进。这导致 Stage 3 的模型几乎无法学习新知识，只能在极小范围内微调。

### 2.3 梯度与权重更新健康度

#### 2.3.1 正则化手段全景

| 正则化手段 | Stage 1 | Stage 2 | Stage 3 | 作用机制 |
|-----------|---------|---------|---------|---------|
| Weight Decay | 0.1 | 0.1 | 0.1 | L2 正则化，防止权重过大 |
| Label Smoothing | 0.1→0.05 (退火) | 0.05→0.025 (退火) | 0.01 (固定) | 防止过度自信预测 |
| BPE Dropout | 0.1 | 0.1 | 0.0 | 输入增强 |
| Attention Dropout | 0.1 | 0.05 | 0.0 | 注意力矩阵正则化 |
| Focal Loss (γ) | 1.5 | 1.0 | 禁用 | 关注难样本 |
| DropBlock | — | — | 0.1 (size=3) | 结构化注意力 Dropout |
| Stochastic Depth | — | — | 0.05 | 随机层丢弃 |
| Max Grad Norm | 1.0 | 1.0 | 1.0 | 梯度裁剪 |
| EMA Decay | 0.999 | 0.999 | 0.999 | 权重指数移动平均 |

#### 2.3.2 EMA 的量化贡献

| 阶段 | Base Val Loss | EMA Val Loss | 改善量 | 改善率 |
|------|--------------|-------------|--------|--------|
| Stage 1 (CLM) | 3.7647 | 3.6766 | -0.0881 | **-2.3%** (Loss) / **8.4%** (PPL) |
| Stage 2 (MNTP) | 3.7245 | 3.6554 | -0.0691 | **-1.9%** (Loss) / **6.7%** (PPL) |
| Stage 3 (Polish) | 3.6989 | 3.6909 | -0.0080 | **-0.2%** (Loss) / **0.8%** (PPL) |

**诊断**: EMA 的贡献呈现 **显著递减效应**：
- Stage 1: EMA 改善 8.4% PPL，因为高 LR (6e-4) + 高 Dropout (0.1) 产生较大梯度噪声，EMA 平滑效果显著
- Stage 2: EMA 改善 6.7% PPL，LR 降低但仍有一定噪声
- Stage 3: EMA 仅改善 0.8% PPL，极低 LR (2e-5) + 无 Dropout 下，梯度噪声已很小，EMA 几乎无贡献

#### 2.3.3 Gradient Clipping 分析

所有阶段均使用 `max_grad_norm=1.0`。在 bf16 混合精度训练中，梯度裁剪是必要的安全机制。对于 94.2M 参数的模型，1.0 的阈值是标准选择，既不会过度限制梯度更新，也能防止梯度爆炸。

| 阶段 | 模型参数 | 优化器状态 | 梯度 | 激活值 | 总计/GPU |
|------|---------|-----------|------|--------|---------|
| Stage 1 | ~188MB (bf16) | ~752MB (fp32) | ~188MB | ~28GB | ~32GB |
| Stage 2 | ~188MB | ~752MB | ~188MB | ~28GB | ~32GB |
| Stage 3 | ~188MB | ~752MB | ~188MB | ~28GB | ~32GB |

*注: 激活值估算基于 seq_len=1024, batch_size=16, 14 层 Transformer, gradient checkpointing 启用*

48GB A6000 显存中约 32GB 被使用，**显存利用率约 66.7%**，剩余 ~16GB 余量。

#### 5.1.3 算力利用率

```
理论 FLOPs/步 ≈ 6 × params × seq_len × batch_size
  = 6 × 94.2M × 1024 × 32
  ≈ 18.5 TFLOPs/步

实际吞吐: 2.3 步/秒 (加权平均)
  → 实际算力: ~42.6 TFLOPs/秒

A6000 理论 FP16 算力: 38.7 TFLOPs × 4 = 154.8 TFLOPs
  → GPU 利用率: 42.6 / 154.8 ≈ 27.5%
```

**诊断**: GPU 利用率仅约 **27.5%**，存在显著优化空间。主要瓶颈：
1. **数据加载 I/O**: 训练数据存储在机械硬盘上，读取速度受限
2. **梯度同步通信**: 4 GPU DDP 通信开销
3. **Gradient Checkpointing**: 以时间换空间，降低了计算密度
4. **数据预处理**: CPU 端的 tokenization 和数据增强

## 第六章：根因分析与高阶优化建议

### 6.1 根因分析

#### 问题 1: PPL 改善进入瓶颈期

**现象**: V12→V13 仅改善 0.26 PPL（38.94→38.68），参数量增加 74%（54M→94M）

**根因链**:
```
数据量硬限制 (100M tokens)
    ↓
tokens/param = 1.1× (远低于理想 20-200×)
    ↓
模型容量大量闲置
    ↓
增加参数 → 几乎无 PPL 改善
    ↓
Stage 3 Polish 正则化过强 → 反而退化
```

#### 问题 2: 语法能力习得不充分

**现象**: ZhoBLiMP 63.47%，3 个维度低于随机基线

**根因链**:
```
训练数据中语法对比样本不足
    ↓
CLM/MNTP 目标不直接优化语法判断
    ↓
模型依赖浅层模式匹配 (100% 任务 vs 0% 任务的极端分化)
    ↓
深层语法推理能力缺失 (anaphor, passive, npi_licensing)
```

#### 问题 3: Stage 3 Polish 无效

**现象**: Stage 3 EMA PPL (40.08) > Stage 2 EMA PPL (38.68)

**根因链**:
```
LR 从 5e-4 骤降至 2e-5 (25倍衰减)
    ↓
模型几乎无法学习新知识
    ↓
DropBlock (0.1) + StochDepth (0.05) 引入额外噪声
    ↓
Val Loss 从 Epoch 1 开始持续恶化
    ↓
Stage 3 EMA 反而不如 Stage 2 EMA
```

### 6.2 高阶优化建议

#### 优先级 1: 缩小模型 + 增加数据效率 (预期 PPL 改善: 1-3 点)

| 策略 | 当前 | 建议 | 预期效果 |
|------|------|------|---------|
| 缩小 d_model | 768 | 576-640 | tokens/param 提升至 1.8-2.2× |
| 减少层数 | 14 | 12 | 减少计算量，改善梯度流 |
| 减少词表 | 32,000 | 16,000-24,000 | 减少嵌入参数占比 |
| **目标参数量** | **94.2M** | **45-55M** | **匹配数据规模** |

#### 优先级 2: 数据增强 (预期 PPL 改善: 2-5 点)

| 策略 | 具体方法 | 实施难度 | 预期效果 |
|------|---------|---------|---------|
| **语法模式上采样** | 对 ZhoBLiMP 弱维度相关文本上采样 1.5-2× | 低 | 语法能力 +3-5% |
| **PPL 过滤阈值调整** | 当前 max_ppl=200, 放宽至 300-500 | 低 | 保留更多有价值的低频文本 |
| **领域平衡** | 确保对话/文学/学术/新闻多领域覆盖 | 中 | 泛化能力提升 |
| **合成数据** | 使用大模型生成高质量中文文本 | 中 | 有效数据量 +20-50% |

#### 优先级 3: 训练策略优化 (预期 PPL 改善: 0.5-2 点)

| 策略 | 当前 | 建议 | 理由 |
|------|------|------|------|
| **移除 Stage 3** | 3 阶段 | 2 阶段 (CLM+MNTP) | Stage 3 已证明无效 |
| **LR 精调** | 6e-4→5e-4→2e-5 | 尝试 3e-4~1e-3 grid search | 找最优 LR |
| **增大 batch** | 16×2=32 | 24×2=48 或 32×2=64 | 更稳定梯度估计 |
| **多轮自蒸馏** | 0 轮 | 2-3 轮迭代 | 每轮改善 0.1-0.3 PPL |
| **课程学习** | 无 | 先简单后复杂文本 | 加速收敛 |

#### 优先级 4: 正则化优化 (预期 PPL 改善: 0.5-1 点)

| 策略 | 当前 | 建议 | 理由 |
|------|------|------|------|
| Focal Loss γ | 1.5→1.0→0 | 尝试固定 γ=0.5-1.0 | 避免退火过快 |
| Label Smoothing | 退火 0.1→0 | 尝试固定 0.05 | 保持后期正则化 |
| DropBlock | Stage 3 才引入 | 从 Stage 1 引入 size=5 | 更早的结构化正则 |
| Dropout 调度 | 固定→0 | 线性退火 0.2→0.05 | 更平滑过渡 |

#### 优先级 5: 架构改进 (预期 PPL 改善: 1-3 点)

| 改进 | 描述 | 风险 |
|------|------|------|
| **Flash Attention 2** | 替换 SDPA，训练速度 +20-30% | 兼容性 |
| **RoPE 基频调整** | θ=10000→500000 | 长序列能力提升 |
| **分层学习率** | 底层 LR 低，顶层 LR 高 | 底层通用表征更稳定 |
| **嵌入空间 Mixup** | 在 embedding 层引入混合增强 | 正则化效果 |

### 6.3 推荐方案

#### 方案 A: 最优 PPL 追求 (推荐)

```
架构: LLaMA, d_model=640, n_layer=12, n_head=10, n_kv=5
  → ~55M params, tokens/param ≈ 1.8×

数据: PPL 过滤 (max_ppl=300) + 语法模式上采样 1.5×
  → 有效训练数据增加 ~30%

训练管线 (2 阶段):
  Stage 1: CLM + SGDR + Focal Loss (γ=1.0) + EMA
    → 10 epochs, lr=5e-4, batch=24×2=48
  Stage 2: MNTP + Dynamic CLM + EMA
    → 12 epochs, lr=4e-4, patience=10

预期: PPL ≈ 37.5-38.0, ZhoBLiMP ≈ 64-66%
```

#### 方案 B: 最优参数效率

```
架构: 复用 V11 (d_model=512, 12L, 38.7M params)
  → tokens/param ≈ 2.6× (最佳比例)

训练: 3 阶段 (CLM → MNTP → Self-Distill Round 2)
  + 数据重加权 + 更激进 Focal Loss (γ=2.5)

预期: PPL ≈ 39.5-40.0, 但参数效率最优
```

#### 方案 C: 集成策略

```
模型: V12 EMA + V13 EMA 权重平均
  → (state_dict_v12 + state_dict_v13) / 2

预期: PPL 可能降至 37-38 范围
风险: 需要两个模型架构一致，当前 V12(54M) ≠ V13(94M)
```
